# Code Principles

- Minimize scope. Prefer the simplest correct diff.
- Follow existing conventions and surrounding patterns.
- Avoid over-engineering and premature abstraction.
- Prefer naming, types, and structure over explanatory comments.
- Add tests only when they materially validate the changed behavior.
