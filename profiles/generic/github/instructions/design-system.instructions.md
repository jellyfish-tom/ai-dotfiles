# Design System Guidance

- Reuse the target repo's existing component library, design tokens, and CSS variables before introducing new styling primitives.
- Check existing icons, assets, wrappers, and shared UI primitives before adding custom alternatives.
- Follow the repository's established visual language and layout patterns instead of introducing parallel systems.
- If the repo ships a project-specific design-system instruction layer, prefer that over this generic baseline.
